# Contributing

Thanks for your interest in improving pub_doctor.

## Setup

```bash
git clone https://github.com/Sahad2701/pub_doctor
cd pub_doctor
dart pub get
dart test
```

## Making changes

* Keep changes small and focused.
* Run formatting before committing:

```bash
dart format .
```

* Ensure no analysis issues:

```bash
dart analyze
```

* Add tests for new features or fixes.
* Update `CHANGELOG.md` if behavior changes.

---

## Pull Requests

* One feature or fix per PR.
* Describe what changed and why.
* Make sure CI passes.

---

## Reporting Issues

Open an issue with:

* pub_doctor version
* Dart SDK version
* OS details
* Steps to reproduce
* Expected vs actual behavior
